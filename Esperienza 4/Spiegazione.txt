IN PILLOLE

Dobbiamo creare una rete che abbia dei router (piu' di uno) , in ognuno questi installiamo quagga
Quagga è una suite di programmi per il network routing
Tra questi abbiamo Open Shortest Path First (OSPF Open Shortest Path Fist è un protocollo di routing di IP per trovare il 
percorso piu' breve), Routing Information Protocol (RIP) e Border Gateway Protocol (BGP)

Con quagga la tua macchina scambia informazioni di routing con altri router , utilizzando protocolli di routing
.Quagga utilizza queste informazioni per aggiornare le tabelle di routing del kernel.
Puoi cambiare dinamicamente la configurazione e puoi vedere le informazioni della tabella di routing dall'interfaccia 
a terminale di quagga.Un sistema con quagga installato si comporta come un router dedicato
Quagga è costituito da un insieme di demoni,che implementano i vari protocolli di routing.
Le informazioni ottenute dai vari protocolli sono “raggruppate” da un demone specifico (Zebra) che
ha il compito di comunicare con il kernel del PC

Per ottenere i demoni sopra descritti bisogna installare il software Quagga
Per mandare in esecuzione uno qualsiasi dei demoni relativi ai
protocolli di routing bisogna scrivere un file di configurazione

I file di configurazione sono file di testo in cui sono riportati dei
comandi che permettono di configurare il protocollo OSPF secondo le proprie esigenzenze

Esempi di file di configurazione si possono trovare nella cartella
/etc/quagga/


INSTALLAZIONE QUAGGA

apt-get install quagga

#copia dei file configurazione *.conf.sample nella cartella etc/quagga

CONFIGURAZIONE BGPD


In telecomunicazioni e informatica il Border Gateway Protocol (BGP) è un protocollo di routing usato per connettere tra 
loro più router che appartengono a sistemi autonomi (Autonomous System, AS) distinti e che vengono chiamati router 
gateway o router di bordo/confine. È quindi un protocollo di routing inter-AS, nonostante possa essere utilizzato anche 
tra router appartenenti allo stesso AS (nel qual caso è indicato con il nome di iBGP, Interior Border Gateway Protocol), 
o tra router connessi tramite un ulteriore AS che li separa.



Prima di tutto dobbiamo configurare il router con BGP 
lanciamo:

router bgp asn

#router bgp 10

Dove Autonomous System number (asn) è un numero che scegliamo
Dopo avere settato il router con BGP ,siamo in grado di lanciare i comandi BGP

bgp router-id A.B.C.D
#bgp router-id 1.1.1.1

Setta l'ID del router bgp

network A.B.C.D/M

Questo comando dice che la rete A.B.C.D/M sarà annunciata a tutti i vicini

neighbor peer next-hop-self

Indica il punto di uscita da un AS
Dove è che i pacchetti saranno istradati

neighbor peer remote-as asn

Creiamo un nuovo vicino il quale ha un ASN come vicino


neighbor 172.16.1.2 remote-as 10


Questo comando specifica l' AS number di un vicino.
Questo numero è usato per determinare se il vicino è un router BGP interno o esterno
Se l'AS number ha lo stesso As number dell'attuale BGP configurazione , il vicino è un BGP router interno
Invece , se gli AS numbers sono differenti, il vicino è un BGP router esterno.
In questo caso , stiamo dicendo al nostro router che l'indirizzo 172.16.1.2 si trova nel nostro
stesso AS number. Poichè il nostro router sta anche in AS 10

distance 30 172.16.1.0/24 

Assegnamo un peso al link che collega il nostro attuale router con la rete di indirizzo 172.16.1.0/24
Assegnamo 30
